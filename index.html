ER диаграмма для сдачи отчетности использования лекарственных средств.

Процесс автоматизации сбора данных

Логика - Каждая мед организация сдает отчет по своему району, район в область, область в РЦЭЗ. Начинается все с того что работник мед организации заходит в личный кабинет где ему предоставляется веб форма для заполнения отчетности. После заполнения работник нажимает кнопку "Отправить отчет", затем данный отчет появляется уже в личном кабинете у района. Сотрудник района проверив отчет и не обнаружив ошибок нажимает кнопку "Одобрить отчет". Если имеются ошибки в отчете то сотрудник района нажимает кнопку "На исправление" и в появившемся поле указывает причину отклонения одобрения. После одобрения районом данный отчет появляется в личном кабинете у области и далее по такой же логике у РЦЭЗ.

  • Статус отчета записывается в базу и отображается в личном кабинете.
  • При каждой отправке отчета, а так же при смене статуса одобрения происходит email уведомление (это видно в диаграмме). 
Используемый стэк описан ниже под диаграммой



У каждой области должен иметься свой код ID для записи в БД. 
При создании отчета так же должен создаваться автоинкрементируемый ID. В итоге при сдаче отчета должен отображаться уникальный код отчета, типа [Код обл + Код отчета + Дата] - таким образом на любом этапе отчета можно сразу понять чей это отчет и когда был сделан.
Пример: 001-218-14-02-2026 

Стек при небольших нагрузках - 
Хранение - для хранения данных можно выбрать даже обычный MySql 
Бэк - можно написать на PHP
Фронт - Bootstrap 
Сервер - достаточно виртуального хоста с панелью Plesk, PS вполне может подойти
Анализ данных - старый добрый Бобёр (DBeaver)
Визуализация - PowerBI, Tableau

Стек при больших нагрузках - 
Хранение - Postgre 
Бэк - можно написать на PHP
Фронт - Bootstrap 
Сервер - VPS
Анализ данных - старый добрый Бобёр (DBeaver)
Визуализация - PowerBI, Tableau
